ARRI File

 

Purpose

 

This node will load ARRI RAW files and set their properties before passing them to other nodes. Supports .mxf, .ari, and .arx files.

Supports NVIDIA hardware acceleration (Select File -> Preferences-> Codec -> Arri -> Decoder Type = Cuda for faster processing.

 

Inputs

 

Raw: Input to connect ARRI files coming from other nodes. (Alternatively you can drag & drop an ARRI file on top of this node to define the source files)

 

 

Outputs

 

AV Out: The ARRI RAW files specified in the File property, with the metadata settings for the image extraction defined in this node.

Node specific properties

 

Container type:  Indicates if it is an enumerated sequences (.ari, .arx) or a movie file (.mxf)

 

Debayer (From File, Custom):

 

When "From File" mode is selected the debayering settings will be loaded from the input files, using the native ARRI Log-C Wide Gamut by default. So it is not necessary to select ARRI Log-C in Custom mode (and neither possible. if you want ARRI Log-C, just leave it in From Input mode )

 

The next options will only appear when selecting the Custom option.

 

Color Space: This particular property is ARRI specific, but depending on the selection it will also set the gamma and gamut properties of the node, and they will be displayed below. When using the native ARRI Log-C just leave it in From Input.

 

Change ISO: Activate this check box and a field will appear where you can set the desired ISO value.

 

Change Kelvin: Activate this check box and a field will appear where you can set the desired color temperature.

 

Change Tint: Activate this check box and a field will appear where you can set the desired Tint value.

 

Sharpness: Here you can assign a numerical value to set the sharpness level.

 

Reduce Factor: Extract the image at a reduced resolution, given by the reduction factor. To increase decoding performance when the images will not be used at their  maximum resolution.

 

More detailed information about ARRI RAW settings can be found here.

 

 

Troubleshooting

 

When using the CUDA decoder for ARRI files (File->Preferences->Codecs->ARRI decoder), if the ARRI decoder refuses to start with an error related to NVIDIA (or CUDA):

 

Install the latest NVidia driver supported for your GPU.  NVIDIA encoders are rapidly changing, and having an obsolete NVIDIA driver is a common cause of failure.

 

If you are using remote desktops to launch Mistika Workflows (or virtual machines), make sure that the graphic session has a valid CUDA and openGL context. In particular, the basic version of Microsoft RDP is known to destroy the graphics context on disconnection.

 

You can also select a different ARRI decoder different to CUDA,  (File->Preferences->Codecs->ARRI decoder), at least to help on troubleshooting.
