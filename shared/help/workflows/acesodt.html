ACES ODT

 

Purpose

 

Applies an ACES color transformation, typically when from camera RAW formats to other color spaces, according to ACES-ODT standard.

Inputs

 

Input_1:  Any media files.

 

Outputs

 

Output: The input media files alongside with the defined color transformation

 

Note that this node will not render any images by itself. What it does is to provide the color transformation to posterior nodes with render capabilities.

 

Standard properties

 

Object Name

Color

Bypass

 

Node specific properties

 

Input -> Gamma, Gamut:  Color space of the input media.

Input -> Use Bradford Matrix: Use Bradford Matrix algorithm when decoding the input images.

Output -> Color Space: Select the color space you want for the output, which will determine the output Gamma & Gamut.

Output -> Precision: Select the algorithm to be used for the transformation (either faster or more accurate options but slower)

Output -> Scale:  

No Scaling: No changes

Scale from RGB444 SMPTE range, it will scale from video levels to data levels. Simplifying, that means expanding values from the 64...940 range  into the 0...1023 range (in 10bit notation).  This is commonly (but wrongly) referred as "Full Range". Improper selection of this  transformation will produce increased contrast and even out of range values.
