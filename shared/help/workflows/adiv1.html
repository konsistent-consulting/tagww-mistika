ADI v1, SkyADI

 

Purpose

 

This node creates an XML file in ADI v1.1 / Cablelabs 1.1 format (or ADIv3 / Cablelabs 3.0 in the case of SkyADI).

The  ADI standard  (Asset Distribution Interface)  is used by many broadcasters  for creating complex delivery packages with structured content and metadata. And it is based in Cablelabs metadata specifications. The delivery package is typically made of an XML file alongside the file structure that it describes (xml file, video files, subtitle files, and icon/still files).

This node is not intended to be used manually. It is provided to facilitate programmers the integration with their asset management systems when in the need to produce ADI compliant deliveries.

A typical ADI delivery workflow requires three nodes, and this node provides the middle part  of the workflow, as follows:

 

1 - A metadata import node:  A custom node thating the properties of the ADI nodes.  Logically this node depends on each asset management system, so it needs to be developed for kseach specific case.

 

2 - ADI node. This  node validates metadata coherence with the media files and creates an ADI xml file according to the ADIv1 specification.

 

3 - Upload node:  This node will get  all the media files and the ADI xml file and upload the whole package to the destination servers. It may be based on existing nodes (tar, aspera, etc) or it may also require additional customization.

 

In general, nodes 1 and 3 need to be developed to the specific requirements of each broadcaster by their own programmers.  Alternatively, SGO also offers services for custom developments of this type.

 

Inputs, Outputs, Properties.

 

As per the ADIv1.1 / Cablelabs 1.1 standard (or ADIv3 / Cablelabs 3.0 for the case of the  SkyADI node, which is a particular case of ADIv3  for this broadcaster).
