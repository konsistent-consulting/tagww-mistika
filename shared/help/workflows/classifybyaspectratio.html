Classify By Aspect Ratio

 

 

Purpose

 

This python node will classify the input media by its aspect ratio, sending different ranges of aspect ratios trough different connectors

 

As a difference to other metadata classifiers, this node provides a Threshold property to match images to the "most similar" aspect ratio rather than forcing to strict matching.

 

Inputs

 

files: To connect a files to be classified. You can connect a Folder or a Watcher node, for example.

 

Outputs

 

Output List of possible aspect ratios: A definable list of connectors. You can add/remove connectors by adding/removing them on the Output List property,

 

other: Input media files not matching any aspect ratio in the Output List (plus/minus a defined threshold) will be sent trough this connector

 

notFound: Input files with no resolution metadata properly defined (aspect ratio not found) will be sent trough this connector.

Node specific properties

 

Output List: Comma separated list of aspect ratio options (decimal places accepted). By default:  0.56,1.00,1.33,1.77,1.85,2,2.35. You can modify the list to your likes, and the connectors will be refreshed immediately.

 

Threshold:  Percentage of difference permitted for aspect ratio matching.  Note that if more than one matches are possible this node will only send the media file trough the first matching connector.

 

Error mode:  When no aspect ratio matching is found, this property defines what kind of error message will be sent to the error log (None, Info, Warning, Critical). This is mainly to facilitate user scripts parsing render log files later (or human monitoring). But note that this is just a log message, the case Critical will not stop the workflow execution because this node provides its own error control branches via the notFound and other connectors.
