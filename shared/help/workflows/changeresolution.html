Change Resolution

 

Purpose

 

This node allows to redefine the render resolution of the workflow, either by a scale factor (percentage) or in absolute number of pixels, then passing the new resolution specifications downstream.

Note that the node will not render a new version by itself, instead it will define the new resolution to be passed as metadata downstream, to tell posterior transcoder nodes with render capability to render the file at the new resolution

IMPORTANT NOTE: Please read the dstNode property below to understand how it works. Otherwise you may encounter unexpected results.

 

Inputs

 

File: File folder or content from another node.

 

Outputs

Node specific properties

 

dstNode: Optional. It permits to specify the name of a posterior node that will render the new resolution, and from that point the resolution will stay the same in posterior nodes. This may be necessary because this node has the unique capability to select relative scaling rather than absolute. This is a powerful feature, but logically you may want to avoid every node downstream applying another relative rescaling again and again.

 

For example, Let's say there is an incoming clip that happens to be 4K and you have defined a scaling percentage of 50%.  if there are 3 transcoder nodes connected downstream one after the other then they will all receive the 50% scaling request by default, so the first one will render at 2K and pass a 2K output to the next, which will then apply another 50% (rendering at 1K), and the third one will render at 0.5K.  If you want to avoid this behavior then you can simply specify the name of the destination node for the resolution change request.

In that way, all nodes connected at the output downstream will render at the original resolution initially until reaching the dstNode, which will render at 50% (2K in this example), and from that node the resolution will stay at 2K for all posterior nodes.

 

Mode:

 

Percentage change the resolution in percentage. You can select this percentage in the scaleFactor field. (See the above explanation for more control...).

 

Absolute changes horizontal and vertical resolution of the output images. You can select the most common resolution presets (HD, UHD, etc). The "Same as input" option keeps the input media resolution. "Custom" allows to set a custom resolution.

 

ForceEvenRes: It will force rounding to an even resolution (recommended, otherwise odd resolutions can create troubles to codecs based on YU422 and other sampling rates requiring an even number of pixels)

 

Warning: Most formats have their own resolution limits, an error will appear at execution time if their limits are exceeded.
