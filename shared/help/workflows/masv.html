MASV

 

Purpose

 

Upload content to a Massive.io platform account and generate the corresponding download links.

Note: To access Massive.io you need to have created an account on this platform. In addition, you will need to have a API Key to access your account from the Workflows node.

It is also required to indicate your Team ID, which can be retrieved from the browser's url once logged in in your massive.io account:

 

 

 

 

 

 

Inputs

 

Files: files to be uploaded

 

Outputs

 

links: package download link.

These links are embedded in the Universal Path of each input file as privateData, so it can be included in the body of any notification's node with a python code injection:

Node specific properties

 

 

 

Package Name: Name of the uploaded package. Dynamic metadata tokens can be used here.

 

Description: Description of the uploaded package. Dynamic metadata tokens can be used here.

 

Recipients: email addresses where MASV notification of the uploaded package will be sent.

 

Package Link: Toggle to create a package download link.

 

Team ID: ID of massive.io account, it can be retrieved from the browser's url once logged in.

 

API Key: Key generated in the massive.io account to be used.

 

Chunk Size: MB of data to be used per each uploaded fragment. Optimal values will depend on the available bandwidth.

 

Base API URL: API version to be used.

 

Check Render log and Script Editor panels for execution details.
