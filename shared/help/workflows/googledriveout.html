Google Drive Out

 

Purpose

 

This node upload incoming files to a Google Drive folder

Before you can use this node you will need to get some credentials from Google. It only needs to be done one time, but the node will not work until then:

Warning:  After following the below procedure and save the workflows file the new 

Mistika Workflows  .mwf file will contain your credentials and long term token. 

Anyone having access to the  .mwf file can use the associated google services in your account.

For more safety or if not sure, create a dedicated google account for your workflows, not personal accounts.

 

We can't go into much detail (screenshots and so on) because Google change this procedure quite often, but basically it works as follows. Once you have a Google Drive:

1 - Obtain Google OAuth 2.0 cloud credentials for your project. In general your Google cloud credentials are managed in the google cloud console in this web page:

https://console.cloud.google.com/apis/credentials/oauthclient

 

If your never used the google console you will need to use the Create new project menu to create a new project (as a project name, for example name it "Mistika Workflows").

 

Having your project selected, go to Enable APIS and Services and select the google resources you want to give access to Mistika Workflows (YouTube API, google drive API, Gmail,...), and click Enable on each one.

 

2 - Then go to Credentials and:

 

- Configure consent screen: Follow Google instructions to select if you want internal or external access from your organization. Then fill the form with your own email (both the support email and developer email).  And for the app name just type Mistika Workflows. At the date of this document the rest of the fields are optional and can be left by default. When it is done press Save and continue.

 

- Credentials: After the above step, go to Credentials->Create Credentials -> Oauth client ID and select ApplicationType = Desktop App, Name = Mistika Workflows. Then click Create. By doing that you will get Client ID and Client Secret credentials that you can then use in this node. Click Save to confirm and exit.

 

 

3 -  Once in Mistika Workflows, put the credentials in the node properties clientID / clientSecret, you need to authenticate a first time. Depending on the Workflows version the Authenticate action may appear as a property of the node, or in the contextual menu of the node, or by executing the workflow a first time. When it is executed for the first time a new web browser will appear where you have to login one time to the Google account (the account giving access to the google resources you want to access).  

 

If successfully done an "authentication successful"  or "The authentication flow has completed" message will appear.  This means the node is ready to use (depending on the node more properties may appear in the properties panel for additional configuration of your Google resource).

 

Each node can have the same credentials or be different to access different Google accounts and resources

Node specific properties

 

Client ID, Client Secret: Access credentials for the Google Drive.

 

Folder ID: The ID of the Google Drive folder to receive the files. To get the ID, navigate to your Google drive folder manually and then copy/paste the last part of the url to this property (the string after the last "/" character).

 

Example. If the url in your browser is this:

 

https://drive.google.com/drive/u/0/folders/1hifq1yft3Gqu86uqasdaV8BCFkhEwlis0

 

Then copy/paste this string: 1hifq1yft3Gqu86uqasdaV8BCFkhEwlis0
