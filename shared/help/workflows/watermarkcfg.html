Watermark Cfg

 

 

Purpose

 

To configure user text for the Watermark display filter,  providing properties to type your own (head, body, and footer), which can then be passed to transcoder task nodes.

 

Mistika display filters are effect templates created in Mistika Boutique or Mistika Ultima. In general, in Mistika Workflows you only need to select the display filter in any transcoder node to use  it. But when the display filter contains text strings (such as the particular case of the Watermark display filter) then you may want to configure it with your own text at workflow execution time, which is what this node does.

 

For example, if you want to apply watermark text to some media files and render the results to ProRes you can create a workflow with these nodes:

 

Media Files -> Watermark Cfg -> ProRes  (with the ProRes node having its display filter property set to "Watermark")

 

This node is designed to be used with the "Watermark" Display Fiilter (which comes pre-installed here:

 

User_home_folder/SGO AppData/shared/display_filters/watermark.dfilt

 

It can only change the three metadata strings, so if you want to change the sizes, positions, or the  "look" of the text you will need to modify the Watermark display filter in Mistika Boutique.

 

Note: This node does not "render" any images by itself. What it does is to configure the display filter with the new metadata. Then, posterior transcoder nodes will create the necessary text vectors and render the images.

 

A newer & more complex  version of this node is the CSV To DFILT node, which can read the three text strings from a .CSV file.

 

Note: This node can only replace the text used by a display filter, if what you want is to replace the image files used in a display filter then use the setDisplayFilterFiles node instead.

 

Inputs

 

In: Any media files

 

Outputs

 

out: The input media files, plus the text metadata to be used at render time.
