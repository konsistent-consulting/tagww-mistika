Dolby Vision

 

Purpose

 

To apply Dolby Vision tone mapping.

This node gets a master media file containing an HDR master (in any supported image format) having a Dolby Vision .xml  sidecar companion file (that is, a .xml metadata file created with Dolby Vision software, with the same name and in the same folder as the media file but with .xml extension), and it uses Dolby Vision to create a new version for the selected target (a target display capable of a lower number of nits).

This node does not render new media directly. Instead, you will need to connect this node to any transcoder node downstream, which will render the actual media file.

This node uses a Dolby Vision plugin, so it requires to have a Dolby Vision license installed in the system.

 

The xml sidecar file needs to have ".xml" as second extension, to unequivocally identify the file. Examples:

HDR Master: MyHDRmaster.mxf

Sidecar file: MyHDRmaster.mxf.xml

 

In that way, if there are several versions ( MyHDRmaster.mxf, MyHDRmaster.mov, etc ) Mistika Workflows will know which sidecar file belongs to which media, (MyHDRmaster.mxf in this particular example)

 

When using enumerated sequences just remove the frame number. Example:

HDR Master: MyHDRmaster_00001.exr  ... MyHDRmaster_00580.exr

Sidecar file: MyHDRmaster.exr.xml

 

 

Inputs

 

Input_1: Input media files containing the HDR master.

 

Outputs

 

Output: The media files with the Dolby Vision tone mapping applied, ready to be rendered in a posterior transcoder node.
