Hard Link

 

Purpose

 

This node creates a hard link of the input files to the desired path destination.

 

This can be useful in certain situations where we have all the original media in one folder location, but we do not want to move it or  make a copy of it.

Creating a hard link offers an efficient solution, treating the link as the original file without consuming additional storage space. It is ideal for scenarios such as consolidating media in an EDL/XML/AAF to a new folder or initiating Watch Folder workflow by receiving the hard linked file.  

It is highly efficient as the link is treated as if it was the original file, and it does not increase the storage usage.

 

To understand the difference between hard links and soft links, check out this article.

 

 

 

It is important to note that:

- A hard link will only work if the destination path is located in the same filesystem where the original file to be linked is.

- Modifying a hard link will modify the original file as well.

- As the node uses Python libraries to create the link, only Linux and Windows are supported. It will not work in a MacOS environment.

 

 

Inputs

 

Files: The source files to create the links from.

 

 

Outputs

 

Links: File path of the links created.
