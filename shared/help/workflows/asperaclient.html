Aspera Client

 

Purpose

 

File transfer through the Aspera platform. This node allows you to upload or download files (depending if you use the Input or Output node versions).

 

Outputs (input nodes only)

 

To: Files downloaded from the Aspera Server.

 

Inputs (output nodes only)

 

File: Files to be uploaded to Aspera.

 

 

Standard properties

Object Name

Color

Lock

 

Node specific properties

 

User: Type here the username of your account.

 

Certificate:  You can login by using a ssl cartificate credentials. Locate the certificate by clicking on the "..." button.

 

Password: Type the password to access to your account.

 

Local file path (Input nodes only): Local file path is a property for the input Aspera node where you have to assign the folder where the downloaded files will be placed.

 

Remote file path: It is the server address and it must contain the complete path, including folders and subfolders where the content to download is located (Input node version), or the exact location where you want to upload the files (Output node version).

 

TCP port: TCP port, if it is specific for connection.

 

UDP port: UDP port, if it is specific for connection.

 

Target Rate (Mbps): You can indicate the target bitrate.

 

Retries: Is the name of times that workflows try to upload to aspera the files if the uploading fails. So, if the uploading fails, Workflows will retry to upload the files the indicated number of times

 

Rename Files List: You can list the characters you want to change and the characters to supply them. The first character is the character you want to change and the second character is the character to supply it. So, for example, if you want to change the "a" characters in the file name to "e" just fill the field with "ae" (without quotes).  Also, you can change more than one character. To do this, separate it with commas. So, to change  the "a" to "e" and the "u" to "g": "ae,ug" without quotes.
