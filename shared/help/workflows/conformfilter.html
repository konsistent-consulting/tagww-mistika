Conform Filter

 

Purpose

 

To filter the input media files by selecting the ones that are mentioned (or not mentioned) in an EDL, XML, or AAF file. The node will output the files included in the EDL/XML/AAF file trough the included connector and all the others trough the excluded connector.

For example you could use this node to backup or copy the files that are included in an offline edition and delete all the others.

Here there is a video tutorial explaining the Conform Filter

 

Inputs

 

EDL/XML/AAF: File to be conformed, containing a list of edit decisions in one of the supported standards (EDL, XML, or AAF).

 

In: Any number of file folders containing media files (typically Folder nodes, which can have the recursive property active to process all subfolders recursively.  It accepts multiple In connections (they will appear automatically when connecting more nodes).

 

Outputs

 

Include, exclude:  Input files are sent trough one or the other connector depending if they are used or not in the EDL/XML/AAF

 

Standard properties

Object Name

Color

Bypass

Lock

 

Node specific properties

 

 

File Path: Path to the EDL, XML, or AAF file containing the list of cuts, as per each of the corresponding standards. Leave it empty to get it from the  EDL/XML/AAF connector.

 

FPS: Frame rate for the conform.

 

Match by (Tape Name, Clip name): To indicate which metadata field is used for the conform, the Tape Name or the Clip Name.

 

Match by path: If active, the full paths mentioned in the editing list will be used, otherwise only the filenames are matched.

 

Match Extension: If active, the file extension will be used when searching for matching files. Typically you will want to deactivate it when conforming against multiple file formats, and you will want to activate it when having different versions of the same clip but in different formats (to ensure using the format that is specifically mentioned in the EDL/XML/AAF)

 

No trim: The whole media files are sent trough output connectors without trimming.

 

Handles: Number of extra head & tail frames to be added to the matching clips. When passing those clips to a transcoder node downstream it will render the segments between the time codes specified in the EDL/XML/AAF, plus this number of extra frames at the head and tail of the clips.
